## Method

- 3 embedding types (TF-IDF, MiniLM, KaLM) x 3 clustering algorithms (K-Means, GMM, HDBSCAN) = 9 combinations
- UMAP dimensionality reduction with hyperparameter tuning
- Evaluated with silhouette, Davies-Bouldin, Calinski-Harabasz, and topic coherence (c_v)
- Validated with ARI/NMI against ArXiv categories and cross-embedding agreement
- Growth trends tested with linear regression, sensitivity analysis, and seasonality normalization

## Project Structure

```
notebooks/
  01_data_collection.ipynb          — ArXiv API collection (181k papers)
  02_exploratory_analysis.ipynb     — EDA, category and term distributions
  03_preprocessing.ipynb            — Text cleaning, survey flagging, embeddings
  04_dimensionality_reduction.ipynb — PCA/SVD baseline, UMAP reduction and tuning
  05_clustering.ipynb               — K-Means, GMM, HDBSCAN comparison
  06_validation.ipynb               — Noise analysis, ARI/NMI, cross-embedding agreement
  07_trends_and_insights.ipynb      — Growth analysis, business recommendations

src/
  collect.py       — ArXiv API data collection with checkpointing
  preprocess.py    — Text cleaning, filtering, lemmatization
  features.py      — TF-IDF, embeddings, UMAP, topic coherence
  label.py         — Cluster labeling via Claude Opus API

data/              — raw and processed data (gitignored, regenerated by pipeline)
figures/           — exported visualizations for report
report/            — final case study PDF
```

## Reproducibility

1. Clone this repo
2. Create a Python 3.11 environment: `conda env create -f environment.yml`
3. Activate: `conda activate arxiv-trends`
4. Install dependencies: `pip install -r requirements.txt`
5. Download spaCy model: `python -m spacy download en_core_web_sm`
6. Set `ANTHROPIC_API_KEY` env var (needed for cluster labeling only)
7. Run notebooks 01-07 in order

**Hardware note:** Full pipeline requires a CUDA GPU for sentence-transformer embeddings. KaLM embedding takes ~2 hours, MiniLM ~5 minutes. UMAP runs take ~15-30 minutes each.
